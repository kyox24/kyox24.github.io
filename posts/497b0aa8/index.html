ML-Agents学习笔记（二）3DBall实例之运行训练好的Model

发表于2020-10-01|更新于2021-05-04|学习笔记
|阅读量:

原文：Ml-Agents项目的Doc/Getting Started Guide

本文上传于2020-10-1

运行已经训练好的Model

这一部分同样因为我对Unity的操作比较熟悉，关于Unity的操作就不会过于赘述了。

添加项目

在Unity Hub，添加下载好的ML-Agents文件夹中的Project然后就能打开了。

打开场景

打开Assets/ML-Agents/Examples/3DBall/Scenes/3DBall。

然后就能看到3DBall的场景了。

理解Unity的环境

实际上只要打开项目后，点击Play按钮就能让场景动起来并看到训练好的Agent是如何采取行动了。但是为了更加深刻地理解这个案例，在实际运行场景之前还是决定简单说明一下3DBall这一案例。

上图可以大致分为两个主体，靠下的蓝色方块头和上边的小球。这个案例开始运行时，上方的的小球会落到蓝色方块头上，则我们希望我们的Agent可以控制蓝色方块头，来尽可能地避免小球从头顶掉落。

Agent

刚刚也提到了Agent这个词，所谓Agent在强化学习的语境下就是动作的执行者。我们让Agent在环境中进行Exploitation和Exploration以来达到学习的目的。老实说我对这两个词的真正含义以及这两件事情之间的Trade-Off都是一知半解，未来进行学习之后应该会就这个问题再写一篇文章。

在本文所提及的3DBall的案例中，我们在上图的Hierarchy视图中的任一3DBall的GameObject中就能找到Agent。Agent同样也是一个GameObject，我们可以看到它挂载了下图中四个脚本：

目前也还没有对这四个脚本进行过多的研究，但是能预想到未来是肯定需要对这几个脚本进行一定程度的研究的。在这里姑且先只介绍原文中提到的一些东西。

Behavior Parameters

从名字上来看，也就是关于Agent采取行动的参数，从某种程度上，可以认为这一脚本决定了Agent如何行动。

上图便是一些参数了，下面对上图中红框部分进行简单说明：

  • Vector Observation-Space Size：
    • Observation代表的是Agent对环境观察后得到的信息，换句话说就是Agent对环境的了解。
    • Vector Observation代表的是这些观察而来的信息所组成的向量。
    • Space Size就自然是这个向量所在的向量空间的维数，这里的维数可以简单的理解成环境中有几个要素是需要Agent在意的。
    • 在这个案例中Observation的维数是8，分别是蓝色方块头的x与z方向上的旋转角度，以及关于小球的x、y、z方向上的相对位置与速度。
  • Vector Action-Space Size：
    • Action代表的是Agent在环境中可采取的行动。
    • 其余的与上面说的十分类似，就不再赘述了，在这个案例中Action的维数是2，分别是蓝色方块头的x与z方向上的旋转角度。
    • 说明了Agent将会通过蓝色方块头的x与z方向上的旋转角度来避免小球从头顶掉落。

运行事先训练好的Model

虽然前面也说过了其实直接点击Play 按钮就可以，但是官方的教程却多写了几步，我也姑且整理出来。

找到3DBall的Prefab

在Assets/ML-Agents/Examples/3DBall/Prefabs你就能找到下图中的Prefab，双击左边那个蓝色方块头。

接着就能在Hierarchy 视图中看到3DBall这一Prefab的构成。

接着查看Agent的组件Behavior Parameters (Script)中的Model是否已经挂载了相应的model。

若有的话直接点击Play按钮即可运行，若没有的话，可以到Assets/ML-Agents/Examples/3DBall/TFModels找到相应的Model。

然后挂载到Prefab的Model的地方去即可运行。
