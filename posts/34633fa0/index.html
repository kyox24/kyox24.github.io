Bootstrapping Conditional GANs for Video Game Level Generation

概述

本文主要提出了一种新的GAN架构，Conditional Embedding Self-Attention Generative Adversarial Network（CESAGAN），同时在训练时引入了Bootstrapping，即把生成的可玩关卡加入到训练集中。在实验部分生成了GVGAI框架中的类塞尔达游戏的关卡。

关键点

PCGML的主要难点

  • 生成游戏内容本身这一任务的训练资料远少于其他常见的生成任务
  • 游戏内容有功能性需求，在生成人脸时哪怕少了一只眼睛还是能被我们识别为人脸，但是对于生成的游戏内容，如果根本无法游玩则没有任何意义。解决功能性需求的方法之一是结合PCGML和Search-based的方法

在使用GANs生成游戏关卡时需要做到的事

  1. 减少Discriminator需要的训练数据量
  2. 提高生成内容的质量
  3. 增加生成内容的多样性

本文使用Bootstrapping来应对上述的1，用CESAGAN来应对2和3。

需要CESAGANs的原因

以往的利用GANs进行关卡生成的研究中，使用CNN的占多数，但是CNN中想要让相距比较远的两个像素产生关联需要的是更大的Kernel Size或者神经网络的深度。这个问题在生成游戏关卡的时候将会被放大，可行的方法是用较小Kernel Size和较深神经网络深度进行训练。

用小而深的原因是基于计算量的考量，参见卷积神经网络的卷积核大小、卷积层数、每层map个数都是如何确定下来的呢？

CESAGANs和SAGANs的区别

这里主要介绍损失函数上的区别，为了满足生成关卡的功能性需求，CESAGANs引入了一个新的feature vector，这个vector由一个embedding network进行训练。

SAGANs的损失函数：

SAGANs的损失函数在本论文中被写错了，在本博文中是参见SAGANs的原始论文Self-Attention Generative Adversarial Networks后正确的写法。

LD=−E(x,y)∼pdata[min(0,−1+D(x,y)]−Ez∼px,y∼pdata[min(0,−1−D(G(z),y)]L_D = -\mathbb{E}_{(x,y) \sim p_{data}}\left[min(0,-1+D(x,y)\right]-\mathbb{E}_{z\sim p_x,y \sim p_{data}}\left[min(0,-1-D(G(z),y)\right] LD​=−E(x,y)∼pdata​​[min(0,−1+D(x,y)]−Ez∼px​,y∼pdata​​[min(0,−1−D(G(z),y)]

LG=−Ez∼px,y∼pdataD(G(z),y)L_G=-\mathbb E_{z\sim p_x,y \sim p_{data}}D(G(z),y) LG​=−Ez∼px​,y∼pdata​​D(G(z),y)

因为是损失函数的形式所以我们希望它的值越小越好，这里“越小越好”指的是我们希望损失函数在变小的同时，事情也在往我们期待的状态转移。

  • 对于LDL_DLD​，我们希望的是随着训练进行D(x,y)D(x,y)D(x,y)能变大，D(G(z),y)D(G(z),y)D(G(z),y)能变小，SAGANs构造的LDL_DLD​可以满足需求。
  • 对于LGL_GLG​，我们则希望D(G(z),y)D(G(z),y)D(G(z),y)能变大，所以上面的LGL_GLG​也能满足需求。

CESAGANs的损失函数

LD=−E(x,y)∼pdata[min(0,−1+D((x,y)∣u)]−Ez∼px,y∼pdata[min(0,−1−D((G(z),y)∣u)]L_D = -\mathbb{E}_{(x,y) \sim p_{data}}\left[min(0,-1+D((x,y)|\bold u)\right]-\mathbb{E}_{z\sim p_x,y \sim p_{data}}\left[min(0,-1-D((G(z),y)|\bold u)\right] LD​=−E(x,y)∼pdata​​[min(0,−1+D((x,y)∣u)]−Ez∼px​,y∼pdata​​[min(0,−1−D((G(z),y)∣u)]

LG=−Ez∼px,y∼pdataD((G(z),y)∣u)L_G=-\mathbb E_{z\sim p_x,y \sim p_{data}}D((G(z),y)|\bold u) LG​=−Ez∼px​,y∼pdata​​D((G(z),y)∣u)

u\bold uu即为用embedding network训练的feature vector，在本文中会通过MLP进行训练。

Bootstrapping

指的是每个epoch后，会测试本次epoch中生成的关卡是否满足功能性需求，若满足则加入到训练集中供下一个epoch使用。本文中检测生成关卡的手法是启发式方法，未来也许可以使用基于Agents的方法来检测。

实验

神经网络架构

基本设定

如果没特别说明表示Generator和Discriminator有着相同设定。

  • Kernel Size：1×11 \times 11×1
  • Activation Function：ReLU
  • Batch Size：32
  • Learning Rate：初始为0.00010.00010.0001
  • 使用Batch Normalization
  • 使用RMSprop
  • 使用ASCII码对关卡进行编码
  • 有两套训练集
    1. 含有45个人类设计的关卡
    2. 含有5个人类设计的关卡
  • 生成15000个关卡

检测功能性需求的方法

制定了以下7个标准：

  1. 只有一个玩家出生点
  2. 只有一把钥匙
  3. 只有一扇门
  4. 敌人在地图的覆盖率小于等于60%
  5. 玩家从出生点开始通过A*算法到达钥匙的位置
  6. 玩家从出生点开始通过A*算法到达门的位置
  7. 有用墙围成的完整边界

在这一部分论文的作者再次强调可以不使用这种启发式的方法，可以使用Agent。

实验结果

主要做了以下4组实验：

  1. {GAN, 含有45个人类设计的关卡的训练集}
  2. {CESAGAN, 含有45个人类设计的关卡的训练集}
  3. {GAN, 含有5个人类设计的关卡的训练集}
  4. {CESAGAN+Boostrapping, 含有5个人类设计的关卡的训练集}·

接下来的结果中，可玩关卡和重复关卡的对比主要是说明CESAGAN的基本表现，即生成可玩关卡以及不重复关卡。在后面的分析中主要是说明CESAGAN可以生成多样性相对高的关卡。

可玩关卡和重复关卡的对比

上图：在含有45个人类设计的关卡的训练集上训练后，CESAGAN生成的可玩关卡比GAN多，但是重复关卡没有显著差异。

下图：作者设计在只含有5个人类设计的关卡的训练集上的实验的主要原因是，对于大部分的Video Game来说45个关卡是一个大到不现实的数量。从结果来看，我们可以看到CESAGAN+boostrapping的效果很好。

汉明距离

根据汉明距离的维基百科：在信息论中，两个等长字符串之间的汉明距离（英语：Hamming distance）是两个字符串对应位置的不同字符的个数。换句话说，它就是将一个字符串变换成另外一个字符串所需要替换的字符个数。

地块的分布情况

通过计算可以得知CESAGAN生成的关卡中地块分布的标准差高于GAN接近一倍，这也说明了CESAGAN偏向于产生更多样的关卡。
