代码段[1]

插入程序代码的方式有两种：使用反引号 `` 、使用缩进Tab`。

  • 插入行内代码，即插入一个单词或者一句代码的情况，` code ` 使用这样的形式插入。
  • 插入多行代码，在开头使用三个反引号 ``` 即可 。或者使用缩进。
  • 其中个人使用体验上，在Typora使用缩进方式编辑代码段时比较麻烦。
  • 另外，多行代码在Typora上的语法为以三个反引号 ``` 开头即可，而不是常见的使用其包裹。

特别说明

  • |、-、:之间的多余空格会被忽略，不影响布局。
  • 默认标题栏居中对齐，内容居左对齐。
  • -:表示内容和标题栏居右对齐，:-表示内容和标题栏居左对齐，:-:表示内容和标题栏居中对齐。
  • 内容和|之间的多余空格会被忽略，每行第一个|和最后一个|可以省略，-的数量至少有一个。

Tag Plugins是Hexo独有的功能，并不是标准的Markdown格式。

以下内容大部分直接来自Butterfly作者JerryC的博客。

以下写法只保证适用于Hexo的Butterfly主题。

Tag Plugins虽然能为主题带来一些额外的功能和UI方面的强化，但是也有明显的限制，使用时请留意。

画树图[3]

画出来是下面的样子：

graph TB A((A)) B((B)) C((C)) D((D)) A-->B A-->C A-->D

接下来是说明，以A((B))举例，好比有一个字典 dict{A:B}，A就是key键值，B就是value，在数的圈圈里显示的是value。-->就是指向线，也可以“初始化的时候就赋值”，例如：A((A))-->B((B))其实就等价与A((A))<CR>B((B))<CR>A-->B

代码中的graph TB写成graph Top to Bottom，推而广之就有：

graph BT

graph BT A((A)) B((B)) C((C)) D((D)) A-->B A-->C A-->D

graph LR

graph LR A((A)) B((B)) C((C)) D((D)) A-->B A-->C A-->D

graph RL

graph RL A((A)) B((B)) C((C)) D((D)) A-->B A-->C A-->D
